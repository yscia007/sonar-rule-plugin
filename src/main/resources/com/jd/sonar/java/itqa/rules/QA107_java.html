避免将synchronized锁加在方法上，而是加在方法内部需要同步的代码块上，这样可以避免同步覆盖多余的代码从而影响系统性能。

错误示例

  
    public class A {
        synchronized void    foo2(int value) { }  // Noncompliant
        synchronized Object  foo4(int value) { return null; }  // Noncompliant
    }

正确示例

    
    public class B {
        public void operate2() {
            synchronized (name)  // Compliant
            {
                if (name.equals(""))  {
                    name = "boonya";
                }
            }
        }
    }
